# Bug: `severity_overrides` Not Applied to Cross-Skill Findings

**Upstream:** [cisco-ai-defense/skill-scanner](https://github.com/cisco-ai-defense/skill-scanner)
**Version:** 2.0.1 (commit [`67a29d9`](https://github.com/cisco-ai-defense/skill-scanner/commit/67a29d9e1a54cef87d0dc77a889ec13ac1357f23))
**Date:** 12026-03-07
**Status:** Fixed upstream — merged in [2.0.2](https://github.com/cisco-ai-defense/skill-scanner/releases/tag/2.0.2)
**Upstream PR:** [cisco-ai-defense/skill-scanner#55](https://github.com/cisco-ai-defense/skill-scanner/pull/55) — `fix: apply policy filters to cross-skill findings`

---

## Summary

Policy `severity_overrides` defined in a custom scan policy YAML are not applied to cross-skill findings generated by `scan_directory()` (the `scan-all` code path). Individual skill findings are correctly overridden. Only cross-skill findings produced when `--check-overlap` is active are affected. The `disabled_rules` filter has the same gap.

## Symptom

A policy override demoting `CROSS_SKILL_DATA_RELAY` from HIGH to MEDIUM has no effect:

```yaml
# skill-scanner-policy.yaml
severity_overrides:
  - rule_id: CROSS_SKILL_DATA_RELAY
    severity: MEDIUM
    reason: "In a curated catalog, cross-skill composability is expected"
```

Scanner output still reports:

```
Cross-Skill Findings:
  [HIGH] Potential data relay attack pattern detected
```

The `--fail-on-severity high` exit code also reflects the un-overridden HIGH, causing CI to fail.

## Root Cause

`_apply_severity_overrides()` is called inside `_scan_single_skill()` for individual skill findings, but **never** on cross-skill findings in `scan_directory()`.

### Where overrides ARE applied (individual skills)

[`scanner.py#L282`](https://github.com/cisco-ai-defense/skill-scanner/blob/67a29d9e1a54cef87d0dc77a889ec13ac1357f23/skill_scanner/core/scanner.py#L282):

```python
def _scan_single_skill(self, skill, skill_directory):
    # ... run all analyzers, collect all_findings ...

    # Apply severity overrides from policy
    self._apply_severity_overrides(all_findings)       # <-- applied here

    # ... compute analyzability, normalize, annotate ...
```

### Where overrides are NOT applied (cross-skill findings)

[`scanner.py#L718-L741`](https://github.com/cisco-ai-defense/skill-scanner/blob/67a29d9e1a54cef87d0dc77a889ec13ac1357f23/skill_scanner/core/scanner.py#L718-L741):

```python
def scan_directory(self, skills_directory, recursive, check_overlap, *, lenient):
    # ... scan each skill via _scan_single_skill (overrides applied per-skill) ...

    # Cross-skill analysis
    overlap_findings = []
    cross_findings = []
    if check_overlap and len(loaded_skills) > 1:
        # ...
        cross_findings = cross_analyzer.analyze_skill_set(loaded_skills)

    if overlap_findings or cross_findings:
        all_cross_findings = list(overlap_findings or []) + list(cross_findings or [])
        if all_cross_findings:
            report.add_cross_skill_findings(all_cross_findings)
            # ^-- severity_overrides and disabled_rules never applied
```

### Hardcoded severity in cross-skill scanner

[`cross_skill_scanner.py#L155-L165`](https://github.com/cisco-ai-defense/skill-scanner/blob/67a29d9e1a54cef87d0dc77a889ec13ac1357f23/skill_scanner/core/analyzers/cross_skill_scanner.py#L155-L165):

```python
Finding(
    id=f"CROSS_SKILL_RELAY_{hash(...) & 0xFFFFFFFF:08x}",
    rule_id="CROSS_SKILL_DATA_RELAY",
    severity=Severity.HIGH,          # hardcoded, never overridden
    ...
)
```

### Exit code also affected

[`models.py#L310-L314`](https://github.com/cisco-ai-defense/skill-scanner/blob/67a29d9e1a54cef87d0dc77a889ec13ac1357f23/skill_scanner/core/models.py#L310-L314):

```python
def add_cross_skill_findings(self, findings):
    self.cross_skill_findings.extend(findings)
    self.total_findings += len(findings)
    self._increment_severity_counters(findings)  # counts HIGH before any override
```

The severity counters drive `--fail-on-severity`. Since overrides are never applied, the counter reflects the hardcoded HIGH, and `--fail-on-severity high` exits non-zero regardless of the policy.

## Affected Rule IDs

All cross-skill rule IDs are affected. These are generated by `CrossSkillScanner.analyze_skill_set()` and `SkillScanner._check_description_overlap()`:

| Rule ID                              | Default Severity | Source                                      |
| ------------------------------------ | ---------------- | ------------------------------------------- |
| `CROSS_SKILL_DATA_RELAY`             | HIGH             | `cross_skill_scanner.py`                    |
| `CROSS_SKILL_SHARED_URL`             | MEDIUM           | `cross_skill_scanner.py`                    |
| `CROSS_SKILL_COMPLEMENTARY_TRIGGERS` | LOW              | `cross_skill_scanner.py`                    |
| `CROSS_SKILL_SHARED_PATTERN`         | MEDIUM           | `cross_skill_scanner.py`                    |
| `TRIGGER_OVERLAP_RISK`               | MEDIUM           | `scanner.py` (`_check_description_overlap`) |
| `TRIGGER_OVERLAP_WARNING`            | LOW              | `scanner.py` (`_check_description_overlap`) |

## Fix

Apply `severity_overrides` and `disabled_rules` to cross-skill findings before adding them to the report. In [`scanner.py`](https://github.com/cisco-ai-defense/skill-scanner/blob/67a29d9e1a54cef87d0dc77a889ec13ac1357f23/skill_scanner/core/scanner.py), `scan_directory()` method, after line 740 and before `report.add_cross_skill_findings()`:

```python
        if overlap_findings or cross_findings:
            all_cross_findings = list(overlap_findings or []) + list(cross_findings or [])
            if all_cross_findings:
+               # Apply policy overrides/filters (same as _scan_single_skill)
+               if self.policy.disabled_rules:
+                   all_cross_findings = [
+                       f for f in all_cross_findings
+                       if f.rule_id not in self.policy.disabled_rules
+                   ]
+               self._apply_severity_overrides(all_cross_findings)
                report.add_cross_skill_findings(all_cross_findings)
```

This is consistent with the existing pattern in `_scan_single_skill()` (lines 278-282) where both filters are applied before findings enter the report.

### Test Case

```python
def test_severity_override_applied_to_cross_skill_findings(tmp_path):
    """severity_overrides must apply to cross-skill findings from scan_directory()."""
    from skill_scanner.core.scan_policy import ScanPolicy, SeverityOverride
    from skill_scanner.core.scanner import SkillScanner
    from skill_scanner.core.models import Severity

    # Create two skills that trigger CROSS_SKILL_DATA_RELAY
    _write_collector_skill(tmp_path / "collector")
    _write_exfiltrator_skill(tmp_path / "exfiltrator")

    policy = ScanPolicy.default()
    policy.severity_overrides = [
        SeverityOverride(
            rule_id="CROSS_SKILL_DATA_RELAY",
            severity="MEDIUM",
            reason="testing",
        )
    ]

    scanner = SkillScanner(policy=policy)
    report = scanner.scan_directory(tmp_path, recursive=True, check_overlap=True)

    relay = [f for f in report.cross_skill_findings if f.rule_id == "CROSS_SKILL_DATA_RELAY"]
    assert len(relay) > 0, "Expected CROSS_SKILL_DATA_RELAY finding"
    for f in relay:
        assert f.severity == Severity.MEDIUM, (
            f"Expected MEDIUM after override, got {f.severity}"
        )
```

```python
def test_disabled_rules_applied_to_cross_skill_findings(tmp_path):
    """disabled_rules must suppress cross-skill findings from scan_directory()."""
    from skill_scanner.core.scan_policy import ScanPolicy
    from skill_scanner.core.scanner import SkillScanner

    _write_collector_skill(tmp_path / "collector")
    _write_exfiltrator_skill(tmp_path / "exfiltrator")

    policy = ScanPolicy.default()
    policy.disabled_rules = {"CROSS_SKILL_DATA_RELAY"}

    scanner = SkillScanner(policy=policy)
    report = scanner.scan_directory(tmp_path, recursive=True, check_overlap=True)

    relay = [f for f in report.cross_skill_findings if f.rule_id == "CROSS_SKILL_DATA_RELAY"]
    assert len(relay) == 0, "CROSS_SKILL_DATA_RELAY should be suppressed by disabled_rules"
```

## Impact on This Repository

- **CI:** `--fail-on-severity high` causes workflow failures on full scans (schedule, manual dispatch, PR fallback-to-full) because `CROSS_SKILL_DATA_RELAY` remains at HIGH despite the policy override to MEDIUM.
- **SARIF:** GitHub Code Scanning annotations show incorrect severity for cross-skill findings.
- **Policy file:** [`skill-scanner-policy.yaml`](../../skill-scanner-policy.yaml) lines 508-510 — the override is correctly defined but has no effect.
- **Other overrides:** `BINARY_FILE_DETECTED`, `HIDDEN_DATA_FILE`, and `PYCACHE_FILES_DETECTED` overrides work correctly because they are individual-skill findings processed through `_scan_single_skill()`.
